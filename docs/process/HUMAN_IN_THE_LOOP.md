# Human-in-the-loop（人の判断を組み込む）

自動化・AI エージェント・CI を進めつつ、**人が必ず介在すべき局面** を明示する。本テンプレートでは、人間の介入点を **「PR レビュー承認」 1 点に集約する** 設計を採用する。

## 1. 目的

- 誤った自動マージ、仕様の静かな逸脱、コンプライアンス違反を防ぐ。
- 「誰が・いつ・何を承認するか」を曖昧にしない。
- 同時に、AI エージェントの作業を**中途で止めない**ことで開発スループットを最大化する。

## 2. 設計原則: HITL は PR マージ承認に集約する

従来の「7 カテゴリで停止して人を呼ぶ」設計はエージェントを頻繁に止めるため、本テンプレートでは以下に切り替える：

- **AI エージェントは常に PR を作るところまで完走する**。途中で停止しない。
- **人間の承認ゲートは PR マージのレビューのみ**。CI green と PR 本文の論点提示を確認した上で承認する。
- 重要な判断点は PR 本文の「マージ前確認事項（HITL）」セクションに**明記して提示**する。レビュアはそこをチェックして承認可否を決める。

## 3. PR 本文に明記すべき事項

PR 本文の「マージ前確認事項（HITL）」セクションに、該当があれば必ずチェック項目として書き出す。該当なしの場合も「該当なし」と明記する。

| 論点 | 例 |
|------|---|
| 憲章・SPEC との矛盾 | 要求が文書化された合意と食い違う場合、SPEC 改訂 PR を併出する |
| セキュリティ境界の変更 | 認証・認可・ネットワーク境界・暗号の扱いを変更する |
| 個人情報・機微データの新規取り扱い | 法的・契約上のレビューが必要 |
| 破壊的 API / データ互換の変更 | 利用者・移行コストの説明 |
| 本番設定・インフラの恒久的変更 | 誤操作の影響範囲が大きい |
| 依存関係の大幅更新 | サプライチェーン・ライセンス |
| 高コスト・不可逆な操作 | 例: 大量データ削除、課金トリガー、外部 API への書き込み |

プロジェクトは上表を **削る・増やす** してよい。増やした項目は SPEC または運用手順にリンクする。

## 4. 承認の記録

- 承認は **PR のレビュー承認** に集約する（GitHub の "Approve" ボタン）。
- レビュアは PR 本文の「マージ前確認事項（HITL）」セクションを必ず読み、チェック項目を確認する。
- 口頭のみの承認は原則認めない（緊急時は事後に PR コメントへ「短縮した承認」と「恒久対策」を記載する）。

## 5. AI / エージェントとの境界

- エージェントは作業を**完走させて PR を作成**する（中途で止めない）。
- 上記論点を検出した場合、**PR 本文に警告セクションを追加**して人間レビュアに判断を委ねる。
- 人間が「続行」と言っただけでは憲章違反を正当化しない。SPEC または憲章を更新したうえで再度レビューする。
- 詳細は [AGENT_OPERATING_GUIDE.md](./AGENT_OPERATING_GUIDE.md) を参照。

## 6. 緊急時（インシデント）

- インシデント対応で通常ルールを短縮する場合は、**事後** にインシデントレポートへ「短縮した承認」と「恒久対策」を記載する。
- ただし「PR を介さない直接 push」だけは緊急時でも避け、最小限の hotfix PR を切ること。

## 7. エスカレーション経路

- 一次: PR レビュアー（プロジェクトの CODEOWNERS）
- 二次: プロダクトオーナーまたは技術責任者（[PROJECT_CHARTER.md](../charters/PROJECT_CHARTER.md) のロール表に合わせる）

## 8. ツール側の補助（任意）

`.claude/settings.json` などのエージェント設定で、以下の方針を実装できる（[テンプレート](../../.claude/settings.json) 参照）：

- `permissions.allow`: PR 作成、Jira 書き込み、ブランチ保護で守られた branch への push まで許可
- `permissions.deny`: 破壊的操作（force push to protected, reset --hard, repo delete, secrets read/write）のみ拒否
- `permissions.ask`: PR マージ系のみ（`gh pr merge`, `gh pr review --approve`）
